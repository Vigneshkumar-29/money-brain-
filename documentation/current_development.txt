*Refers to existing code that is fragile, hardcoded, or incomplete.*

### 1. "Ghost" Buttons (UI without Logic)
*   **Export Data**: In `charts.tsx`, the **"Export PDF"** and **"Export CSV"** buttons exist visually but do absolutely nothing when pressed.
*   **Notification Settings**: The UI exists, but push notifications (especially on Expo Go) are tricky. The code currently relies on local scheduling, which is good but lacks deep integration for server-side alerts.

### 2. Hardcoded Values (Rigidity)
*   **Currency**: The app is hardcoded to **Indian Rupee (â‚¹)** and `en-IN` formatting throughout the code. It cannot accommodate users from other regions (USD, EUR, etc.).
*   **Categories**: Category names ("Food", "Shopping") and their associated colors/icons are **hardcoded** inside `charts.tsx` and `index.tsx`.
    *   *Problem*: If you want to add a "Gym" category, you have to edit multiple React files. You cannot create custom categories dynamically as a user.

### 3. "Fake" AI Insights
*   The "AI Insights" section in `charts.tsx` is currently just a simple `if/else` logic block (e.g., "If expenses > income, show alert"). There is no actual Artificial Intelligence or deep analysis of spending patterns happening here.

### 4. Charting Limitations
*   The charts are constructed manually using basic `View` rectangles.
    *   *Problem*: They lack interactivity (users cannot touch to see specific values), animations are basic, and the layout may break if data values vary significantly.

### 5. âœ… Offline Vulnerability (FIXED)
*   The app now uses an offline-first architecture with local caching and sync queue.
    *   **Solution Implemented**:
        - `useNetworkStatus` hook monitors connectivity in real-time
        - `offlineQueue` manages pending transactions when offline
        - `TransactionCache` stores transactions locally for offline viewing
        - `OfflineContext` provides offline-aware transaction methods
        - `OfflineBanner` component shows sync status to users
        - Transactions are saved locally first, then synced when online
        - Automatic retry mechanism for failed syncs
